# Style Test Bot

Telegram-бот с диагностическим тестом "Насколько твой стиль усиливает твою мужскую привлекательность?"

Бот не продает напрямую — он прогревает пользователя и ведет его в Telegram-канал.

## Стек

- Python 3.11+
- aiogram 3.x
- python-dotenv
- без базы данных, состояние пользователей хранится в памяти (словарь `user_states`)

## 1. Создание бота через BotFather

1. Открой Telegram, найди **@BotFather**.
2. Напиши `/newbot`.
3. Укажи имя бота (например, "Style Test Bot").
4. Укажи username бота (должен заканчиваться на `bot`, например `style_test_bot`).
5. BotFather выдаст тебе токен вида:
   ```
   123456789:AAExampleTokenStringHere
   ```
6. Сохрани этот токен — он нужен для переменной `BOT_TOKEN`.

## 2. Настройка переменных окружения

1. Скопируй файл `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```
2. Открой `.env` и заполни:
   ```
   BOT_TOKEN=твой_токен_от_botfather
   CHANNEL_LINK=https://t.me/твой_канал
   PERSONAL_LINK=https://t.me/твой_username
   ```

   - `BOT_TOKEN` — токен, полученный от @BotFather.
   - `CHANNEL_LINK` — ссылка на твой Telegram-канал (куда ведет кнопка "Перейти в канал").
   - `PERSONAL_LINK` — ссылка на твои личные сообщения (куда ведет кнопка "Получить личный разбор", например `https://t.me/имя_пользователя`).

   **Важно:** файл `.env` не должен попадать в git/публичные репозитории — он содержит секретный токен.

## 3. Установка зависимостей

Рекомендуется использовать виртуальное окружение:

```bash
python3 -m venv venv
source venv/bin/activate   # на Windows: venv\Scripts\activate

pip install -r requirements.txt
```

## 4. Запуск локально

```bash
python bot.py
```

Если все настроено правильно, в консоли появится сообщение "Бот запускается...".
Открой своего бота в Telegram и отправь `/start`.

## 5. Деплой на Render

1. Залей проект (`bot.py`, `requirements.txt`, `README.md` — **без** `.env`) в репозиторий на GitHub.
2. Зайди на [render.com](https://render.com) и создай новый **Background Worker** (Background Worker, а не Web Service — бот не принимает HTTP-запросы, он работает через polling).
3. Подключи свой репозиторий.
4. Настройки сервиса:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `python bot.py`
5. В разделе **Environment** добавь переменные окружения:
   - `BOT_TOKEN` — твой токен от BotFather
   - `CHANNEL_LINK` — ссылка на канал
   - `PERSONAL_LINK` — ссылка на личные сообщения
6. Нажми **Deploy**. После успешного деплоя бот будет работать 24/7.

## Переменные окружения (сводка)

| Переменная      | Описание                                              | Обязательно |
|-----------------|--------------------------------------------------------|-------------|
| `BOT_TOKEN`     | Токен бота от @BotFather                               | Да          |
| `CHANNEL_LINK`  | Ссылка на Telegram-канал                               | Нет (есть дефолт-заглушка) |
| `PERSONAL_LINK` | Ссылка на личные сообщения для "Получить личный разбор" | Нет (есть дефолт-заглушка) |

## Команды бота

- `/start` — начало, приветствие и кнопка "Пройти тест".
- `/restart` — сбросить прогресс и начать тест заново.

## Логика и структура

- 20 вопросов, по 5 в каждой из 4 категорий (A, B, C, D).
- Каждый ответ дает 0–3 балла.
- В конце считается общий балл (0–60), баллы по категориям (0–15 каждая) и определяется самая слабая зона.
- По общему баллу определяется один из 4 уровней: NPC, Случайный стиль, Мужчина со вкусом, Магнит.
- После результата бот показывает текст по слабой зоне, блок "что ты можешь терять" и финальные кнопки:
  - "Перейти в канал" (`CHANNEL_LINK`)
  - "Получить личный разбор" (`PERSONAL_LINK`)
  - "Пройти заново" (перезапуск теста)

## Хранение состояния

Состояние каждого пользователя хранится в оперативной памяти процесса в словаре `user_states`.
Это значит:
- При перезапуске бота весь прогресс пользователей сбрасывается — это ожидаемо, без БД иначе никак.
- Бот подходит для одного процесса/инстанса. Если потребуется горизонтальное масштабирование — нужна будет внешняя БД или Redis (в текущей версии не используется).
